HiveMQ (Community Edition) — Phase 2

Active configuration
--------------------
- `config.xml` (mounted at `/opt/hivemq/conf/config.xml`) enables **plain MQTT on 1883** (`mqtt-plain`) and **TLS on 8883** (`mqtt-tls`).
- The Java keystore must exist at `config/certs/broker.jks` (generated by `scripts/gen_broker_keystore.sh`). Without it, the TLS listener will fail to start.

Plain-only broker (no keystore)
-------------------------------
If you are not ready for TLS yet, replace the active config with the plain copy:

  cp config/hivemq/conf/config.plain-only.xml config/hivemq/conf/config.xml

Then set the simulator to plain MQTT (`MQTT_USE_TLS=false`, `MQTT_BROKER_PORT=1883` in `.env` or `config/config.yaml`).

TLS-only broker (no plain 1883)
--------------------------------
Edit `config.xml` and remove the `<tcp-listener>` block for port 1883, or bind plain 1883 to `127.0.0.1` only. Leave `mqtt-tls` on 8883 for remote clients.

File RBAC extension
-------------------
The image is built from `Dockerfile.hivemq`, which installs the **HiveMQ File RBAC** extension and copies the repo’s `config/hivemq/extensions/hivemq-file-rbac-extension/conf/config.xml` (PLAIN passwords, listeners `mqtt-plain` and `mqtt-tls`).

Per-node users and roles are generated into:

  config/hivemq/extensions/hivemq-file-rbac-extension/conf/credentials.xml

by:

  python scripts/generate_campus_secrets.py

That file is **gitignored**; Compose bind-mounts it into the extension. Run the generator before `docker compose up` (or the mount may point at a missing file).

`acl-enterprise-reference.xml` remains a **reference** for Enterprise-style ACLs; CE uses the File RBAC extension instead.
